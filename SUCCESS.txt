HOW IT WORKS:
  1. SQLite requests page N
  2. zstd_vfs decompresses ONLY that page (~4-16 KB)
  3. Returns page to SQLite
  4. Query continues normally
  
  Result: 1TB database compressed to 500GB can be queried without loading 
          either 1TB or 500GB into memory. Only needed pages (~10KB) are 
          decompressed per query.

PRODUCTION READY:

LIMITATIONS TO NOTE:
  ⚠️  EXCLUSIVE locking (one writer at a time)
  ⚠️  No WAL mode support
  ⚠️  Linux/Unix x86-64 primarily
  ⚠️  Test thoroughly before production

EXAMPLE USAGE:
  # Create compressed database
  sqlite3 source.db -bail \
    -cmd '.load ./sqlite_zstd_vfs/build/zstd_vfs.so' \
    "VACUUM INTO 'file:compressed.db?vfs=zstd&level=6'"
  
  # Query it
  sqlite3 :memory: -bail \
    -cmd '.load ./sqlite_zstd_vfs/build/zstd_vfs.so' \
    -cmd ".open 'file:compressed.db?mode=ro&vfs=zstd'" \
    "SELECT * FROM users WHERE id = 123"
